# Workflows Quick Reference

## Side-by-Side Comparison

### 📋 Auto-Deploy Workflow
**File**: `.github/workflows/deploy.yml`

```yaml
on:
  push:
    branches: [main]
  pull_request:
    branches: [main]
```

| Aspect | Details |
|--------|---------|
| **Trigger** | Every push to `main` OR pull request |
| **When** | Automatic (no user action) |
| **Time** | 2-5 minutes after push |
| **Best For** | Regular deployments |
| **Inputs** | None |
| **Output** | Standard logs |

**Use Case**: Normal development workflow
```bash
git push origin main
# Workflow triggers automatically
# Site updates 2-5 minutes later
```

---

### 🚀 Manual Deploy Workflow
**File**: `.github/workflows/manual-deploy.yml`

```yaml
on:
  workflow_dispatch:
    inputs:
      reason:
        type: string
        default: 'Manual deployment'
      clear_cache:
        type: boolean
        default: false
```

| Aspect | Details |
|--------|---------|
| **Trigger** | Click button in GitHub Actions |
| **When** | On-demand (user initiated) |
| **Time** | Immediate when you click |
| **Best For** | Force builds, testing, hotfixes |
| **Inputs** | Reason (text), Clear cache (checkbox) |
| **Output** | Detailed logs + analysis |

**Use Case**: Force updates without code changes
```
1. Go to GitHub Actions
2. Click "🚀 Manual Build & Deploy"
3. Click "Run workflow"
4. Add reason (optional)
5. Workflow runs immediately
```

## When to Use Which?

### Use Auto-Deploy (push code)
```
✓ Making code changes
✓ Updating event data
✓ Fixing bugs
✓ Adding features
✓ Normal development

Just push to main:
$ git push origin main
```

### Use Manual Deploy (click button)
```
✓ Force sitemap update (new API events)
✓ Clear cache and rebuild
✓ Test deployment without code
✓ Deploy hotfixes immediately
✓ Verify workflow works

Go to Actions → Run workflow
```

## Workflow Files

### `.github/workflows/deploy.yml`
**Auto-Deploy on Push**
- Triggers: `on: [push, pull_request]`
- Steps: checkout → setup Node → install → build → deploy
- Output: Standard logs
- File size: ~1.5 KB

### `.github/workflows/manual-deploy.yml`
**Manual Deploy on Demand**
- Triggers: `on: workflow_dispatch`
- Steps: Same as auto + detailed analysis
- Inputs: reason, clear_cache
- Output: Detailed logs + build analysis
- File size: ~5 KB

## Command Reference

### Git Commands
```bash
# Regular push (triggers auto-deploy)
git push origin main

# Make empty commit (triggers rebuild for sitemap)
git commit --allow-empty -m "chore: update sitemap"
git push origin main
```

### GitHub CLI
```bash
# Trigger manual workflow (simple)
gh workflow run manual-deploy.yml

# Trigger with reason
gh workflow run manual-deploy.yml \
  -f reason="Update sitemap"

# Trigger with cache clear
gh workflow run manual-deploy.yml \
  -f clear_cache=true

# List all workflows
gh workflow list
```

### Checking Results
```bash
# Count sitemap URLs
curl https://yosintv.github.io/eventos/sitemap.xml | grep -c "<url>"

# Test homepage
curl https://yosintv.github.io/eventos/ | head -20

# Check robots.txt
curl https://yosintv.github.io/eventos/robots.txt
```

## Common Scenarios

### Scenario 1: Regular Update
```
Action:  git push origin main
Trigger: Auto-deploy workflow
Build:   Builds, fetches API, generates sitemap
Result:  Site live in 2-5 minutes
```

### Scenario 2: Force Sitemap Update
```
Action:  Go to Actions → Manual workflow → Run
Trigger: Manual-deploy workflow
Build:   Fetches latest API data, regenerates
Result:  Sitemap updated immediately
```

### Scenario 3: Clear Cache Rebuild
```
Action:  Manual workflow with "Clear cache" ✓
Trigger: Manual-deploy workflow (cache clean)
Build:   Fresh npm install, full rebuild
Result:  Clean build deployed
```

### Scenario 4: Emergency Hotfix
```
Action:  Manual workflow → Run (no code push!)
Trigger: Manual-deploy workflow immediately
Build:   Uses current code, deploys
Result:  Live in 2-5 minutes without commit
```
